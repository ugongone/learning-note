2. オープン標準の基礎知識

今回の標準化によって何が変わったのか、基礎用語で解説します。

重要キーワードと変化

Agent Skills

AIへの「仕事道具セット」

AIに特定の業務を行わせるための機能パッケージ。（例：GitHub操作、AWS管理など）

従来 毎回プロンプトで長い手順を指示するか、専用スクリプトを書く必要があった。
変化 「スキル」をポン付けするだけで、複雑な手順をすぐに実行可能になった。

MCP

AIとツールをつなぐ「共通規格」

メーカーが異なるAI同士でも同じツールを使えるようにする接続ルール。

従来 Claude用、OpenAI用など、AIごとに接続コードを書き直す必要があった。
変化 共通規格に統一され、1つ作れば主要なAIモデル全てで使い回せるようになった。
今回のニュースの核心：
これまでバラバラだった規格が統一され、「一度作ったスキルは、ClaudeでもChatGPTでもCopilotでも使える」ようになったことです。

エンジニア向け：実装のポイント

Agent Skillsの実体は、「MCPサーバー」そのものです。
既存のMCPツールの知識があれば、呼び出し方も設定ファイルの書き方も全く同じです。 

// claude_desktop_config.json の例
{
  "mcpServers": {
    // 既存のMCPツール（書き方は同じ）
    "filesystem": { "command": "npx", "args": [...] },

    // Agent Skills（実体はMCPサーバー）
    "github-skill": { "command": "npx", "args": [...] } 
  }
}
