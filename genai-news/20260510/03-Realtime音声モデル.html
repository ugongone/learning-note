2026/05/07 発表 — Realtime API は同時に GA

推論する音声、通訳する音声、文字起こす音声

GPT-Realtime-2 / Translate / Whisper の3モデル同時投入と、Realtime APIのGA移行

概要

OpenAIは2026年5月7日、リアルタイム音声向けに3つの新モデル — 推論・対話を担う「GPT-Realtime-2」、70言語以上から13言語へ同時通訳する「GPT-Realtime-Translate」、低遅延ストリーミング文字起こし「GPT-Realtime-Whisper」を同時公開した。 合わせて長らくbetaだったRealtime APIが正式GAに移行し、本番利用に必要なSLAやサポートが整った。GPT-Realtime-2はコンテキスト長を128Kへ拡大し、Big Bench Audioで96.6%（旧81.4%）を達成、音声エージェント開発の事実上の標準モデルへ躍り出た。

3モデル比較

推論・通訳・文字起こしを担う3モデルが、それぞれ役割と価格帯で住み分けられている。

GPT-Realtime-2

Voice + Reasoning

用途: 双方向の音声エージェント
コンテキスト: 32K → 128K
推論: minimal〜xhighの5段階
Big Bench Audio: 96.6%（旧81.4%）
価格: $32/M入力, $64/M出力

GPT-Realtime-Translate

Live Translation

用途: 同時通訳
入力: 70+ 言語
出力: 13 言語
特徴: 会話推論なしで低レイテンシ
価格: $0.034 / 分

GPT-Realtime-Whisper

Streaming Transcription

用途: ライブ字幕・議事録
特徴: レイテンシを制御可能
対象: 会議・コンタクトセンター
価格: $0.017 / 分

Realtime API GA の意味

「人と話すように話せるAI」を 音声入力 → テキスト → 音声出力 という従来の3段パイプラインなしに直接構築するためのAPI。これまではbetaで本番採用が見送られていた領域が、SLA・サポート付きで一気に解禁された。

Realtime APIは音声を WebSocket／WebRTC でストリーミング双方向に やり取りし、モデルが音声のまま考えて、音声のまま返すためのインターフェース。「電話越し」「会議越し」のような連続的な対話を、低レイテンシのまま処理できる。

何に使われるか（典型ユースケース）

コールセンター／音声エージェント

顧客と 普通の電話のテンポで会話 しながら、CRMや在庫APIを 会話中にツール呼び出し して回答する。

リアルタイム同時通訳

国際会議や接客で、話者の声に被せず 数百ms遅れ で別言語の音声を返す。70+→13言語に対応。

ライブ字幕／議事録

配信・会議の音声をストリーミングで文字起こしし、字幕表示と要約を 同時並行 で生成。

車載／ハンズフリーアシスタント

運転中の音声リクエストに 割り込み可能 で応答。「あ、やっぱり違うのにして」が自然に通る。

語学・面接練習アプリ

トーン制御で 厳しめ／優しめの面接官 を切替え、相手の発音や言い直しに即時フィードバック。

アクセシビリティ補助

視覚障害者向けに画面状況を 会話で問いかけ可能 に。WebアプリやPC操作を音声で誘導。

今までできなかったこと → できるようになったこと

観点 これまで（STT+LLM+TTSや旧Realtime） 今回のRealtime-2 + API GA
レイテンシ STT→LLM→TTSの直列で 2〜3秒 遅延し、会話が「無線」っぽくなる 音声→音声で直接処理 し、数百ms で応答開始
推論能力 音声モデルはテキストモデルより2〜3世代古いのが常識 GPT-5クラスの推論 を音声側で利用可能、Big Bench Audio 96.6%
割り込み ユーザーが話し始めるとAI音声を 強制カット するだけで、文脈が崩れる 割り込みを 会話の一部として吸収。訂正・差し戻しが自然に進む
ツール呼び出し 会話の合間に 1ツールずつ逐次 呼ぶ必要があり、待ち時間が出る 並列ツール呼び出しに対応。喋りながらAPIを複数叩ける
通訳 専用通訳サービスを別契約。話速やトーンが崩れる 同一API内に Realtime-Translate があり、声色を保ったまま70+→13言語へ
コンテキスト 32Kで、長電話では 過去の発言を忘れる 128K に拡張。長時間ミーティングや連続商談でも保持
トーン／声色 固定の合成音声、感情はTTS側で別調整 トーン制御＋新声 Cedar / Marin。落ち着き／元気などを指示可能
本番運用 beta扱いで SLA無し、企業導入は様子見 GAでSLA・サポート付き、本番システムに採用しやすい
要するに: 「人と話している感」を成立させる低レイテンシ・推論力・割り込み許容・ツール並列・通訳・長文脈の6点が、betaの制約を脱して同時にクリアされた。これにより、これまで「LLM＋音声」を諦めて画面UIに逃げていた業務（電話オペレーション、対面接客、ライブ通訳など）が現実的な実装対象になる。
