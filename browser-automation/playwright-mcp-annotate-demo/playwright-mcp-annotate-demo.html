矢印や吹き出しを画像編集ツールで毎回描き足す。位置合わせも地味に時間を食う。

そこで Playwright MCP ＋ 注釈Skill を組み合わせ、確認も注釈付きエビデンス生成も自動化します。

Playwright MCP とは AIエージェントが実ブラウザを操作できる仕組み

Playwright MCP は、Claude などの AI エージェントが MCP ツール経由で実ブラウザを操作できるようにする仕組みです。ページ遷移・クリック・入力・スクリプト実行・スクショ撮影を、人手を介さず指示だけで実行できます。

本デモは ブラウザ拡張連携モード（Playwright Extension） を使用。お使いの Chrome に拡張を入れるだけで、ログイン済みの実ブラウザをそのまま自動操作できます。

Playwright Extension を Chrome ウェブストアで見る

※ 拡張インストール後、発行されるトークンを MCP 設定に登録すると連携できます。

ここからは実例です。題材として learning-note にダークモード機能を追加し、その動作確認を Playwright MCP で自動操作 → playwright-screenshot-annotate Skill で赤枠・矢印・吹き出しを付けてからスクショ撮影しました。 手作業ゼロで、下のようなチームにそのまま見せられる検証エビデンスが生成されます。

STEP 1
MCPでブラウザ操作
ローカルサーバへ遷移・クリック・リロード
STEP 2
注釈を注入
対象要素のbboxから枠・矢印・吹き出しを生成
STEP 3
スクショ撮影
注釈付きの状態をそのまま保存
STEP 4
クリーンアップ
注釈を除去し次の検証へ（ページは無傷）

▶ まずは通しで、実際の動きをどうぞ

AI への指示だけで、ブラウザ操作・注釈付け・スクショ撮影が進んでいく様子を録画しました。

※ 動画はクリックで再生／ダブルクリックで拡大できます。

1

BEFORE — ライトモード初期状態

ヘッダー右端にダークモード切替ボタン（data-testid="dark-mode-toggle"）を新規追加した。

localhost:8765/index.html
トグルボタンが意図した位置（breadcrumbの右）に追加されていることを確認
2

クリック直後 — ダークモード適用

MCPでトグルをクリック。<html> に dark クラスが付与され、シェル全体が暗転する。

localhost:8765/index.html
ヘッダー / サイドバー / 背景がダーク化し、アイコンが月→太陽へ切替わることを確認
3

再読込後 — ダークモード維持

ページをリロード。localStorage.theme = 'dark' と <head> 内のチラつき防止スクリプトにより、初期表示からダークを維持する。

localhost:8765/index.html
リロード後も html.dark が維持され、設定が永続化されていることを確認

一発では決まらない。そこは「指示して直す」

最初の注釈は吹き出しが窮屈で読みづらいものでした。そこを AI に口頭レベルの指示で伝え、 位置やレイアウトを調整し直しています。完璧な自動生成ではなく、人が要点だけ指示して仕上げる—— 実際のやり取りはこんな雰囲気です。

このデモで使った仕組み

  • Playwright MCPブラウザ自動操作。navigate / click / evaluate / screenshot を MCP ツールとして呼び出し、ブラウザ拡張経由で実ブラウザを操作。
  • 注釈Skillplaywright-screenshot-annotate。要素の実 bounding box から赤枠・矢印・吹き出しを生成。位置がズレず、撮影後は完全に除去されページを汚さない。
  • 検証対象learning-note のダークモード。1クリックで画面全体が変化するため BEFORE/AFTER が明確で、注釈デモの題材として最適。
  • スコープ外側シェル（header / sidebar / 背景）のみダーク対応。iframe内ノートHTMLは対象外（白いまま浮く設計）。
